# 内容策略

## 站点定位

**搜索入口型聚合站**，而非内容托管站。

- 核心价值：「搜索入口 + 资源聚合」
- 本站自有资源 → 直接跳转自己的网盘（用户留存高，体验最佳）
- 站外索引资源 → 外链跳转 + 广告变现（零维护成本，规避版权风险）

---

## 资源来源分级

### 一级：自有资源（最优）
- 亲自整理、上传到自己网盘的资源
- 链接完全可控，可随时替换
- 直接展示下载链接，无需跳转中间页

### 二级：转存资源（需评估）
- 发现站外高质量资源后，转存到自己网盘替换链接
- **适用条件**：资源体量小、版权风险低（学习资料、工具软件等）
- **避免转存**：影视作品、付费课程、有明确版权声明的内容
- 转存后网盘仍可能因内容审查封链，需定期检查有效性

### 三级：站外索引（聚合模式）
- 仅抓取标题、链接、来源站点、分类等元信息
- 不托管任何内容，责任链在原始分享者
- 展示时明确标注「站外资源」，提供免责跳转提示
- 通过广告变现（用户点击时的展示广告或跳转页广告）

---

## 内容补全策略

### 发现内容缺口
- 通过管理后台「无结果搜索词」（noResultQueries）识别用户需求
- 高频无结果词 → 优先补录目标

### 补全优先级
1. **快速自录**：直接找到资源，录入本站资源库
2. **站外索引**：抓取站外链接写入外链库，本站无结果时自动 fallback 展示
3. **引导搜索**：无结果时展示「去夸克搜索」等快捷入口，辅助用户自助获取

### 自动化辅助（规划中）
- 定时爬虫脚本：抓取目标站点新增资源，写入 `data/external_links.json`
- 管理后台外链库管理：增删查、标记状态（仅外链 / 已转存）
- 搜索 fallback：本站 0 结果时自动从外链库补充展示

---

## 变现逻辑

| 资源类型 | 用户行为 | 变现方式 |
|----------|----------|----------|
| 自有资源 | 直接获取 | 用户留存、回头客、口碑 |
| 转存资源 | 直接获取 | 同上 |
| 站外索引 | 跳转外站 | 跳转页广告、展示广告 |
| 无结果兜底 | 引导搜索 | 搜索引导页广告 |

---

## 风险控制

### 高风险，避免
- 转存影视作品、付费课程等明确版权内容
- 批量自动转存（容易触发网盘风控）
- 假冒原创内容

### 中风险，需注意
- 转存工具软件（部分有授权限制）
- 站外索引链接的内容质量不可控（过期、违规等）

### 低风险，推荐
- 索引公开分享的学习资料、文档、数据集
- 仅展示元信息 + 外链，不托管内容
- 明确标注来源和免责声明

---

## 技术实现规划

```
data/external_links.json        ← 站外链接库（元信息索引）
data/resources.json             ← 自有资源库
scripts/crawl.ts                ← 爬虫脚本（手动/定时触发）
lib/types.ts                    ← ExternalLink 类型定义
lib/store.ts                    ← searchExternalLinks() 函数
pages/api/search.ts             ← 无结果时 fallback 站外库
pages/search.tsx                ← 区分展示「本站资源」/「站外资源」
pages/admin/resources.tsx       ← 外链库管理 Tab
```

### 搜索展示逻辑
- 本站有结果 → 只展示本站资源
- 本站 0 结果 → 展示站外索引资源（带来源标注 + 免责说明）
- 两者都有（可选）→ 本站在前，站外折叠展示
